## Overview

OpenFiskal currently supports one public integration topology: server-to-server.

Your backend holds the API key, adds the seller-scoping header, and calls OpenFiskal on behalf of your locations and registers. Registers, browsers, and mobile apps should never call the API directly.

```mermaid
flowchart LR
    R1["Register 1"] --> B["POS backend"]
    R2["Register 2"] --> B
    R3["Register 3"] --> B
    B -- "Bearer token + X-OpenFiskal-Seller" --> O["OpenFiskal API"]
```

## Not supported

The public API does not currently support device-direct authentication or direct calls from POS clients. If you have an edge-device use case, route it through your backend for now.
