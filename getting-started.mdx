## What's the same across countries

These mechanics are identical regardless of the country guide you follow:

- **API key format** вЂ” `of_{env}_{country}_{random}`, one key per country + environment.
- **Request shape** вЂ” bearer auth on every request, `X-OpenFiskal-Merchant` on every merchant-scoped request, ETag + `If-Match` on mutating calls.
- **Resource ordering** вЂ” merchant в†’ location в†’ register в†’ fiscalize в†’ operation в†’ complete.
- **Idempotency** вЂ” `Idempotency-Key` on `POST /operations` and `PATCH /operations/{id}/complete|void`.
- **Money fields** вЂ” decimal strings, not integers. `pretax_amount + tax_amount + tip_amount = total_amount`.
