## Overview

This is the OpenFiskal Fiscalization API.

Use this page to confirm commercial and compliance fit before you start implementation.

## Supported product scope

This API is currently built for server-to-server communication.

Right now, OpenFiskal does not support direct device-direct communication yet. Your backend must call the API on behalf of your POS devices.

## Supported countries and regimes

The current public scope covers Germany, Austria, and Italy.

| Country | Fiscalization regime | Receipt requirement |
| --- | --- | --- | 
| 🇩🇪 Germany | KassenSichV | You must render all required fiscal fields exactly as returned. |
| 🇦🇹 Austria | RKSV  | You must preserve fiscal values and local formatting requirements. | 
| 🇮🇹 Italy | RT  | You must render the DCW-number onto your receipt. | 

<Warning>
  This matrix is a product scope guide, not legal advice. Your implementation owner should review the regime-specific onboarding checklist before go-live.
</Warning>
