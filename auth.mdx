## API key creation

Create API keys in the tenant dashboard. Each key is tenant-scoped and country-scoped.

<Warning>
  Store the key in your secrets manager. Treat it like any other production secret.
</Warning>

## Use the standard authorization header

Although the credential is an API key, the API uses the standard HTTP bearer pattern:

```bash
-H "Authorization: Bearer ofk_live_abc123..."
```

This is preferable to a custom auth header because it works better with standard API tooling, gateways, SDKs, proxies, and security middleware.

## Organization scoping

For every organization-scoped request, also send:

```bash
-H "X-OpenFiskal-Organization: org_01HXYZ"
```

Together, a typical organization-scoped request looks like this:

```bash
curl https://api.openfiskal.com/v1/operations \
  -H "Authorization: Bearer ofk_live_abc123..." \
  -H "X-OpenFiskal-Organization: org_01HXYZ"
```

Organization-scoped resources include:

- Locations
- Registers
- Register fiscalizations
- Operations
- Exports

You do not need `X-OpenFiskal-Organization` when you create, update, or delete the organization itself.

## Why this design

- Authentication uses a standard header instead of a custom one.
- Merchant context stays explicit on every scoped request.
- The same API key can operate across multiple organizations inside the same tenant and country scope.

## Idempotency

For mutating requests, especially operation start and operation completion, send an `Idempotency-Key` header so retries remain safe:

```bash
curl -X POST https://api.openfiskal.com/v1/operations \
  -H "Authorization: Bearer ofk_live_abc123..." \
  -H "X-OpenFiskal-Organization: org_01HXYZ" \
  -H "Idempotency-Key: op-order-1001-start" \
  -H "Content-Type: application/json" \
  -d '{ "...": "..." }'
```

## Recommended handling

- Reject organization-scoped requests that are missing either `Authorization` or `X-OpenFiskal-Organization`.
- Return `401` when authentication is missing or invalid.
- Return `403` when the API key is valid but not allowed to access the requested tenant, country, or organization.
