## Overview

A register is the logical checkout or terminal that produces operations inside a location. In the current public API, a register stores identity, routing, and compliance metadata that you must keep stable over time.

## Register states

Registers have two public states:

```
[active] ──► [inactive]
```

- `active` means your backend can use the register to create operations
- `inactive` means the register remains in the system for auditability, but your backend should stop creating new operations for it

## Commissioning a register

Create the register after you already know the target organization and location:

```bash
curl -X POST https://api.openfiskal.com/v1/registers \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-OpenFiskal-Organization: org_01HXYZ" \
  -H "Content-Type: application/json" \
  -d '{
    "location_id": "loc_01HXYZ",
    "country_code": "DEU",
    "name": "Kasse 2",
    "external_id": "store-berlin-kasse-2"
  }'
```

## Updating a register

Use `PATCH /registers/{registerId}` when your backend needs to rename the register, update its external mapping, or mark it inactive:

```bash
curl -X PATCH https://api.openfiskal.com/v1/registers/reg_01HXYZ \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-OpenFiskal-Organization: org_01HXYZ" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Kasse 2 - Front Counter",
    "status": "inactive"
  }'
```

## Operational guidance

- Keep `external_id` aligned with the register identifier in your own POS platform
- Keep `terminal_id` or equivalent hardware mapping stable if you use payment terminals
- Capture the operator identity at runtime on the operation, not on the register resource
- Do not recycle a register for a different physical checkout if you need clean audit history
- Prefer inactivation over deletion when you want to preserve historical references

## Listing and lookup

Use `GET /registers` to list registers in the current organization. Use `GET /registers/{registerId}` when you need the current representation for a specific register.
